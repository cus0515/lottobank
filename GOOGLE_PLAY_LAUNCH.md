# LottoBank 구글플레이 출시 준비서

## 현재 상태.

- 개발자 계정 유형은 개인 개발자 계정이다.
- 현재 단계는 Play Console 개발자 신원 확인 진행 중이다.
- 신원 확인 이후 바로 준비할 항목은 앱 생성, 앱 콘텐츠 설문, 폐쇄 테스트, AAB 빌드다.

## 앱 기본 정보.

- 앱 이름. LottoBank.
- 짧은 설명. QR과 인터넷 구매 캡처로 로또 구매 이력과 전적을 관리하세요.
- 긴 설명 초안.

```text
LottoBank는 로또 6/45와 연금복권 720+ 구매 이력을 한 곳에서 관리하는 복권 전적 플랫폼입니다.

QR 코드와 인터넷 구매 캡처를 등록하면 회차, 구매 번호, 당첨 여부, 당첨금, 수익률을 자동으로 정리할 수 있습니다.

주요 기능
- 로또 6/45·연금복권 720+ 당첨번호 조회
- QR 코드 기반 구매 이력 등록
- 인터넷 구매 티켓 캡처 등록
- 나의 구매·당첨·수익률 전적 관리
- 인증 데이터 기반 랭킹
- 번호 레이더와 연구소 백테스터

LottoBank는 동행복권 공식 서비스가 아니며, 당첨 결과 확인은 동행복권 공식 정보를 기준으로 합니다.
```

## 스토어 등록 이미지 준비물.

- 앱 아이콘. 512×512 PNG. 현재 `icon-512.png` 생성 완료.
- PWA 아이콘. 192×192 PNG. 현재 `icon-192.png` 생성 완료.
- 피처 그래픽. 1024×500 PNG. 현재 `feature-graphic.png` 생성 완료.
- 휴대전화 스크린샷. 최소 2장 필요.
- 태블릿 대응을 원하면 7인치, 10인치 스크린샷 추가.

## 앱 콘텐츠 설문 초안.

- 광고 포함 여부. 포함. Google AdSense 사용.
- 앱 접근 권한. 카메라 권한 사용. QR 코드 스캔 목적.
- 위치 권한. 현재 앱에서 실제 위치 저장을 하지 않는 방향이면 사용 안 함으로 유지.
- 계정 생성. 이메일 기반 회원가입 사용.
- 사용자 생성 콘텐츠. 실시간 채팅과 게시글 기능이 있으면 예.
- 개인정보처리방침 URL. `https://lottobank.pages.dev/privacy.html` 생성 완료.
- 금융 상품 여부. 복권 구매 대행이 아니고 기록 관리·정보 조회 서비스로 설명한다.
- 도박/현금성 게임 여부. 실제 베팅이나 구매 대행을 제공하지 않음을 명확히 적는다.

## 데이터 보안 설문 초안.

- 수집 데이터.
  - 이메일 주소.
  - 닉네임.
  - 사용자가 등록한 복권 QR 또는 인터넷 구매 캡처 기반 구매 이력.
  - 채팅 메시지.
  - 앱 사용 기록과 출석 기록.
- 사용 목적.
  - 계정 관리.
  - 구매 이력과 당첨 전적 계산.
  - 랭킹과 서비스 기능 제공.
  - 부정 등록 방지.
- 제3자 공유.
  - Supabase, Cloudflare, Google AdSense처럼 서비스 운영에 필요한 처리자를 명시한다.
- 삭제 요청.
  - 관리자 페이지 또는 문의 이메일을 통해 계정과 등록 데이터 삭제 가능 정책을 준비한다.

## 개인 개발자 폐쇄 테스트 준비.

- 신규 개인 개발자 계정은 production 출시 전 폐쇄 테스트가 필요하다.
- 최소 12명 테스터가 14일 연속 opt-in 상태여야 한다.
- 테스터 이메일 목록 또는 Google Group을 준비한다.
- 테스트 안내문에는 QR 등록, 인터넷 구매 캡처 등록, 회차 조회, 전적, 랭킹, 연구소 기능을 매일 5분 정도 확인하도록 적는다.
- 테스트 피드백 수집 채널을 하나 정한다. 이메일, 카카오톡 오픈채팅, 구글폼 중 하나면 충분하다.

## 기술 구현 방향.

### 1안. TWA 기반 웹앱 래핑.

- 현재 웹앱을 Android Trusted Web Activity로 감싼다.
- 장점. 기존 웹사이트를 거의 그대로 앱화 가능.
- 필요 작업. Android 프로젝트 생성, 패키지명 결정, SHA-256 기반 `assetlinks.json` 배포, AAB 빌드.
- 추천 패키지명. `com.lottobank.app`.

### 2안. Capacitor 기반 WebView 앱.

- 기존 웹앱을 Capacitor로 감싼다.
- 장점. 카메라, 파일 업로드, 푸시 알림 같은 네이티브 기능 확장이 쉽다.
- 단점. Android 프로젝트와 빌드 환경 관리가 필요하다.

## 다음 작업 순서.

1. Play Store용 휴대전화 스크린샷을 만든다.
2. Android 패키지명을 확정한다.
3. TWA 또는 Capacitor 중 하나를 선택한다.
4. AAB 빌드를 생성한다.
5. 내부 테스트 업로드 후 폐쇄 테스트 12명 모집을 시작한다.
6. 14일 테스트 완료 후 production 접근 신청을 작성한다.

## 수익화 가격 정책.

- 1차 출시 상품은 `LottoBank Plus` 단일 구독으로 시작한다.
- 채택 가격은 월 1,500원, 연 14,900원이다.
- Plus 포함 기능은 광고 제거, 연구소 역대 전적 조회 무제한, 번호 저장 확장, 등록 티켓 당첨 알림, 전적 고급 통계다.
- 모임 기능은 초기 상품에서 분리하고, 앱 반응 확인 후 `Group Plus` 월 3,900원 상위 상품으로 검토한다.
- Google Play 앱 내부에서 디지털 기능을 판매할 때는 Google Play Billing 구독 상품으로 연결한다.
- 웹에서는 결제 유도 문구를 과하게 노출하지 않고, 앱 출시 전까지는 “준비 중” 상태로 안내한다.
