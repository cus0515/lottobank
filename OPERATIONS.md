# LottoBank 운영 가이드

## 최초 관리자 설정

1. `admin_operations_setup.sql`을 Supabase SQL Editor에서 실행합니다.
2. 메인 사이트에서 관리자용 이메일로 일반 회원가입을 완료합니다.
3. SQL Editor에서 아래 이메일을 실제 가입 이메일로 바꿔 실행합니다.

```sql
select public.bootstrap_app_admin('admin@example.com');
```

4. `/lotto-control-8f3c2a71.html`에서 관리자 계정으로 로그인합니다.

관리자 주소를 바꾸려면 HTML 파일명을 원하는 값으로 변경한 뒤 다시 배포합니다. 주소를 숨기는 것과 별개로 실제 권한은 `app_admins`에서 검사합니다.

## 데이터 초기화

- `계정 유지 초기화`는 회원과 관리자 계정을 남기고 서비스 데이터를 삭제합니다.
- `회원 포함 전체 초기화`는 Supabase Auth 회원까지 모두 삭제합니다.
- 전체 초기화 후에는 새로 회원가입하고 최초 관리자 설정 절차를 다시 진행합니다.
- 현재 데이터를 처음 한 번 모두 지우려면 SQL Editor에서 `reset_all_data_once.sql`을 실행합니다.

## 일반 운영 순서

1. 운영 상태에서 테이블별 데이터 건수를 확인합니다.
2. 유저, 티켓, 채팅, 게시글 탭에서 필요한 데이터를 조회하거나 삭제합니다.
3. 기능 검증 탭에서 당첨 칭호, 랭킹, 피드 연결을 시험합니다.
4. 문제가 생기면 전체 데이터 탭에서 해당 테이블과 조건만 지정해 수정합니다.

## AdSense 준비

AdSense 승인 전에는 광고 코드를 넣지 않습니다. 승인 후 발급받은 게시자 ID와 광고 단위 ID를 적용합니다.

권장 위치는 다음 세 곳입니다.

1. 데스크톱 중앙 콘텐츠 하단.
2. 모바일 실시간 피드가 끝난 뒤.
3. 회차 조회 결과와 판매점 정보 사이의 독립 영역.

로그인, QR 등록, 번호 확인 버튼 주변과 채팅 입력 영역에는 광고를 배치하지 않습니다.

적용 순서는 다음과 같습니다.

1. AdSense에서 사이트를 추가합니다.
2. 발급된 AdSense 코드를 `index.html`의 `<head>` 안에 넣습니다.
3. 처음에는 자동 광고를 켜고 QR 인증, 로그인, 관리자 페이지는 페이지 제외로 설정합니다.
4. 수동 광고 단위를 사용할 때는 콘텐츠와 충분한 여백을 두고 `광고`로 구분합니다.
5. 게시자 ID를 받은 뒤 루트에 `ads.txt`를 추가합니다.

공식 문서는 아래에서 확인합니다.

- https://support.google.com/adsense/answer/7584263
- https://support.google.com/adsense/answer/9261307
- https://support.google.com/adsense/answer/1346295
